Credenciales Web embebidas

Aplicaciones Windows IIS

Es muy común que los servidores de aplicaciones requieran credenciales para almacenar o recuperar datos, generalmente de un servidor de base de datos. Es muy probable que las aplicaciones alojadas en IIS tengan cadenas de conexión que contengan credenciales para los servidores de bases de datos SQL. De forma predeterminada, estas cadenas de conexión se encuentran en el archivo web.config de la aplicación web como un atributo XML de texto sin formato.

El hecho de dejar credenciales codificadas para cuentas privilegiadas en los archivos de configuración brinda a los atacantes un acceso fácil a información confidencial. Cualquier compromiso del servidor puede llevar al atacante a encontrar estas credenciales y explotarlas para obtener acceso al backend. Además, es probable que esta credencial no cambie con frecuencia debido a los posibles impactos en el servicio, así como a la dificultad de actualizar la configuración de una aplicación web distribuida de múltiples nodos.
