Tareas programadas

Al igual que con los servicios, las credenciales utilizadas por las tareas programadas en Windows también se almacenan localmente para permitir la autenticación continua. Las credenciales para las tareas programadas se almacenan en el servicio del Administrador de credenciales de Windows, al que, como LSA Secrets, puede acceder cualquier persona con privilegios administrativos. Existen muchas herramientas que permiten la recuperación de las contraseñas utilizadas por las tareas programadas, y un atacante puede aprovecharlas fácilmente para descubrir las credenciales que podrían aprovecharse como parte de la cadena de ataques.

Otra vez es importante aplicar el modelo de privilegios mínimos a las tareas programadas. En particular, el 'permiso para que los operadores de servidores programen tareas' debe controlarse estrictamente para evitar la proliferación innecesaria de credenciales. Donde se vayan a ejecutar tareas programadas, los privilegios mínimos se deben aplicar tanto a los administradores del sistema que administren la tarea como a la cuenta utilizada para ejecutar la tarea en sí. Finalmente, la rotación de credenciales nuevamente tiene un papel clave para minimizar el impacto que representa el robo de estas credenciales.
